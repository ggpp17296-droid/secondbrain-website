## 项目初始化

### 1. 创建 Next.js 项目

```bash
cd /Users/jacklee/Desktop/codex_workspace/projects/project-004-secondbrain-demopage/src

# 使用 create-next-app 创建项目
npx create-next-app@latest . --typescript --tailwind --app --src-dir --import-alias "@/*"
```

**选项说明**：
- `--typescript`：使用 TypeScript
- `--tailwind`：使用 Tailwind CSS
- `--app`：使用 App Router（Next.js 13+）
- `--src-dir`：代码放在 src/ 目录
- `--import-alias "@/*"`：设置路径别名

### 2. 安装依赖

```bash
cd src

# 安装核心依赖
npm install gray-matter remark remark-html

# 安装可视化库
npm install markmap-lib markmap-view recharts

# 安装 Headless UI
npm install @headlessui/react

# 安装开发依赖
npm install -D @types/react @types/node
```

### 3. 项目结构调整

```bash
# 创建必要的目录
mkdir -p src/components/{layout,home,learning,projects,knowledge,shared}
mkdir -p src/lib
mkdir -p src/types
mkdir -p src/styles/components
```

## 开发规范

### 代码风格

#### 1. TypeScript 类型定义

```typescript
// ✅ 好的写法
interface Task {
  id: string
  title: string
  status: 'completed' | 'in_progress' | 'pending'
  week: number
}

// ❌ 不好的写法
const task = {
  id: '1',
  title: 'Task',
  status: 'completed',  // 没有类型约束
  week: 1
}
```

#### 2. 组件命名

```typescript
// ✅ 使用 PascalCase
export function ProgressBar() { }
export function TaskCard() { }

// ❌ 不使用 camelCase
export function progressBar() { }
export function taskcard() { }
```

#### 3. 文件命名

```
✅ 组件文件：PascalCase
   - ProgressBar.tsx
   - TaskCard.tsx

✅ 工具文件：camelCase
   - data.ts
   - utils.ts

✅ 页面文件：kebab-case 或 camelCase
   - index.tsx
   - learning.tsx
   - [id].tsx
```

#### 4. 导入顺序

```typescript
// 1. React 和 Next.js
import { useState } from 'react'
import { GetStaticProps } from 'next'

// 2. 第三方库
import { format } from 'date-fns'

// 3. 本地组件
import Layout from '@/components/layout/Layout'
import Hero from '@/components/home/Hero'

// 4. 本地工具
import { loadVaultData } from '@/lib/data'

// 5. 类型定义
import type { VaultData } from '@/types/vault'

// 6. 样式
import styles from './Home.module.css'
```

### 组件开发规范

#### 1. 函数组件

```typescript
// ✅ 推荐写法
interface ProgressBarProps {
  current: number
  total: number
  label?: string
}

export function ProgressBar({ current, total, label }: ProgressBarProps) {
  return (
    <div className="progress-bar">
      {/* ... */}
    </div>
  )
}
```

#### 2. Props 解构

```typescript
// ✅ 解构 props
export function TaskCard({ task, onClick }: TaskCardProps) {
  return <div onClick={onClick}>{task.title}</div>
}

// ❌ 不解构
export function TaskCard(props: TaskCardProps) {
  return <div onClick={props.onClick}>{props.task.title}</div>
}
```

#### 3. 条件渲染

```typescript
// ✅ 使用 && 或三元运算符
{isLoading && <LoadingSpinner />}
{error ? <ErrorMessage /> : <Content />}

// ❌ 使用 if 语句（在 JSX 内）
{if (isLoading) { return <LoadingSpinner /> }}
```

### Git 提交规范

#### Commit Message 格式

```
<type>(<scope>): <subject>

<body>
```

**类型（type）**：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具更新

**示例**：
```bash
git commit -m "feat(learning): add progress bar component"
git commit -m "fix(sync): handle empty daily logs"
git commit -m "docs: update development guide"
```

## 常见问题

### 问题 1：端口被占用

**现象**：
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决**：
```bash
# 方法 1：杀死占用端口的进程
lsof -ti:3000 | xargs kill -9

# 方法 2：使用其他端口
PORT=3001 npm run dev
```

---

### 问题 2：Vault 路径错误

**现象**：
```
Error: ENOENT: no such file or directory, open '/path/to/vault/...'
```

**解决**：
```bash
# 检查环境变量
echo $VAULT_PATH

# 设置正确的路径
export VAULT_PATH=/Users/jacklee/.claude/workspace/my-brain

# 或在 .env.local 中设置
echo "VAULT_PATH=/Users/jacklee/.claude/workspace/my-brain" > .env.local
```

---

### 问题 3：TypeScript 类型错误

**现象**：
```
Type 'string | undefined' is not assignable to type 'string'
```

**解决**：
```typescript
// ❌ 不处理 undefined
const name: string = profile.name

// ✅ 使用可选链和默认值
const name: string = profile.name ?? '未知'

// ✅ 使用类型守卫
if (profile.name) {
  const name: string = profile.name
}
```

---

### 问题 4：样式不生效

**现象**：Tailwind 类名没有生效

**检查清单**：
1. `tailwind.config.js` 的 `content` 是否包含所有组件文件
2. `globals.css` 是否导入了 Tailwind 指令
3. 是否重启了开发服务器

```javascript
// tailwind.config.js
module.exports = {
  content: [
    './src/pages/**/*.{js,ts,jsx,tsx}',
    './src/components/**/*.{js,ts,jsx,tsx}',
  ],
  // ...
}
```

---

### 问题 5：gray-matter 解析错误

**现象**：
```
Error: can not read a block mapping entry; a multiline key may not be an implicit key
```

**原因**：Markdown 文件的 Frontmatter 格式不正确

**解决**：
```yaml
# ❌ 错误格式
---
title: 我的标题
tags: [tag1, tag2]  # 后面有非法字符
---

# ✅ 正确格式
---
title: 我的标题
tags: [tag1, tag2]
---
```

## 性能优化建议

### 1. 图片优化

```typescript
// ✅ 使用 Next.js Image 组件
import Image from 'next/image'

<Image 
  src="/images/profile.jpg" 
  alt="Profile"
  width={200}
  height={200}
  priority  // 首屏图片
/>

// ❌ 使用普通 img 标签
<img src="/images/profile.jpg" alt="Profile" />
```

### 2. 动态导入

```typescript
// ✅ 动态导入大型组件
import dynamic from 'next/dynamic'

const MindMap = dynamic(() => import('@/components/knowledge/MindMap'), {
  loading: () => <LoadingSpinner />,
  ssr: false  // 不在服务端渲染
})
```

### 3. 数据预取

```typescript
// ✅ 使用 getStaticProps 预生成
export const getStaticProps: GetStaticProps = async () => {
  const data = loadVaultData()
  return { props: { data } }
}
```
