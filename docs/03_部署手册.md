## Vercel 部署

### 方式 1：通过 Vercel 网站（推荐）

#### 步骤 1：注册 Vercel

1. 访问 https://vercel.com
2. 使用 GitHub 账号登录
3. 授权 Vercel 访问 GitHub

#### 步骤 2：导入项目

1. 点击 **"Add New..."** → **"Project"**
2. 选择 `secondbrain-demopage` 仓库
3. 点击 **"Import"**

#### 步骤 3：配置项目

**Framework Preset**: Next.js（自动检测）

**Root Directory**: `src`（因为 Next.js 项目在 src/ 子目录）

**Build Command**:
```bash
npm run build
```

**Output Directory**:
```
.next
```

**Install Command**:
```bash
npm install
```

#### 步骤 4：设置环境变量

在 Vercel 项目设置中添加：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `VAULT_PATH` | `./vault` | Vault 路径（GitHub Actions 会克隆到这里） |
| `NODE_VERSION` | `20` | Node.js 版本 |

#### 步骤 5：部署

1. 点击 **"Deploy"**
2. 等待构建完成（约 2-3 分钟）
3. 获得部署 URL：`https://secondbrain-demopage.vercel.app`

## GitHub Actions 配置

### 创建 Workflow 文件

```bash
mkdir -p .github/workflows
```

创建 `.github/workflows/daily-sync.yml`：

```yaml
name: Daily Vault Sync and Deploy

on:
  # 每天北京时间 10:00 (UTC 02:00) 自动运行
  schedule:
    - cron: '0 2 * * *'
  
  # 支持手动触发
  workflow_dispatch:
  
  # 当 Vault 更新时触发（可选）
  repository_dispatch:
    types: [vault-updated]

jobs:
  sync-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
      # 1. 检出网站仓库
      - name: Checkout Website Repo
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}
          fetch-depth: 0
      
      # 2. 检出 Vault 仓库
      - name: Checkout Vault Repo
        uses: actions/checkout@v4
        with:
          repository: your-username/my-brain  # 替换为你的 Vault 仓库
          path: vault
          token: ${{ secrets.VAULT_ACCESS_TOKEN }}
      
      # 3. 设置 Node.js
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: scripts/package-lock.json
      
      # 4. 安装依赖
      - name: Install Dependencies
        run: |
          cd scripts
          npm install
      
      # 5. 运行同步脚本
      - name: Sync Vault Data
        run: node scripts/vault-sync.js
        env:
          VAULT_PATH: ./vault
          OUTPUT_PATH: ./data
      
      # 6. 检查是否有变化
      - name: Check for Changes
        id: changes
        run: |
          if [[ -n $(git status -s data/) ]]; then
            echo "has_changes=true" >> $GITHUB_OUTPUT
          else
            echo "has_changes=false" >> $GITHUB_OUTPUT
          fi
      
      # 7. 提交变化
      - name: Commit Changes
        if: steps.changes.outputs.has_changes == 'true'
        run: |
          git config user.name "GitHub Actions Bot"
          git config user.email "actions@github.com"
          git add data/
          git commit -m "chore: sync vault data $(date +'%Y-%m-%d %H:%M')"
          git push
      
      # 8. 触发 Vercel 部署
      - name: Trigger Vercel Deployment
        if: steps.changes.outputs.has_changes == 'true'
        run: |
          curl -X POST "https://api.vercel.com/v1/integrations/deploy/${{ secrets.VERCEL_DEPLOY_HOOK }}"
      
      # 9. 通知结果（可选）
      - name: Notification
        if: always()
        run: |
          if [[ "${{ steps.changes.outputs.has_changes }}" == "true" ]]; then
            echo "✅ 同步完成并触发部署"
          else
            echo "ℹ️ Vault 无变化，跳过部署"
          fi
```

## 环境变量配置

### GitHub Secrets 配置

在 GitHub 仓库设置中添加 Secrets：

1. 进入仓库 → **Settings** → **Secrets and variables** → **Actions**
2. 点击 **"New repository secret"**

需要添加的 Secrets：

#### 1. VAULT_ACCESS_TOKEN

**用途**：允许 GitHub Actions 访问私有 Vault 仓库

**获取方式**：
1. GitHub 个人设置 → Developer settings → Personal access tokens → Tokens (classic)
2. 点击 "Generate new token (classic)"
3. 设置权限：`repo` (Full control of private repositories)
4. 生成并复制 token

**值**：`ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

#### 2. VERCEL_DEPLOY_HOOK

**用途**：触发 Vercel 重新部署

**获取方式**：
1. Vercel 项目设置 → Git → Deploy Hooks
2. 创建新的 Deploy Hook
3. 复制 URL

**值**：`https://api.vercel.com/v1/integrations/deploy/prj_xxxxxxxxxxxx/xxxxxxxxx`

## 自定义域名

### 添加自定义域名

1. Vercel 项目 → Settings → Domains
2. 输入你的域名，例如：`secondbrain.yourdomain.com`
3. 按照提示配置 DNS

### DNS 配置示例

在你的域名提供商（如 Cloudflare、阿里云）添加：

**类型**：CNAME  
**名称**：secondbrain  
**值**：cname.vercel-dns.com  
**TTL**：Auto

### SSL 证书

Vercel 自动提供免费 SSL 证书（Let's Encrypt），无需额外配置。

## 故障排查

### 问题 1：部署失败

**现象**：Vercel 构建失败

**排查步骤**：
1. 查看 Build Logs
2. 检查是否有 TypeScript 错误
3. 检查依赖是否正确安装
4. 本地运行 `npm run build` 测试

### 问题 2：数据未更新

**现象**：修改 Vault 后网站没有更新

**排查步骤**：
1. 检查 GitHub Actions 是否运行成功
2. 检查 `data/` 目录是否有新的 commit
3. 检查 Vercel 是否有新的部署
4. 检查同步脚本日志

### 问题 3：VAULT_ACCESS_TOKEN 无效

**现象**：GitHub Actions 无法克隆 Vault

**解决**：
1. 重新生成 GitHub Personal Access Token
2. 确保权限包含 `repo`
3. 更新 GitHub Secret
