## 技术栈选型

### 前端框架：Next.js 14+

**选择理由**：
- ✅ **SSG + SSR 混合模式**：首页和知识页用 SSG（快速加载），学习仪表盘用 SSR（实时数据）
- ✅ **文件路由系统**：`pages/index.tsx` 自动映射到 `/`，无需额外配置
- ✅ **API Routes**：`pages/api/sync.ts` 可以做轻量级后端
- ✅ **React 19 兼容**：与现有项目技术栈一致
- ✅ **TypeScript 原生支持**：类型安全
- ✅ **Vercel 一键部署**：零配置 CI/CD

**替代方案对比**：

| 方案 | 优点 | 缺点 | 评分 |
|------|------|------|------|
| **Next.js** | 生态成熟、学习资源多、Vercel 部署简单 | 打包体积略大 | ⭐⭐⭐⭐⭐ |
| Astro | 性能极佳、多框架支持 | 需要重写代码、生态较新 | ⭐⭐⭐⭐ |
| Remix | 现代化、数据加载优雅 | 学习曲线陡、生态小 | ⭐⭐⭐ |
| Gatsby | 插件丰富 | 构建慢、过于复杂 | ⭐⭐ |

**最终选择**：Next.js（综合最优）

---

### UI 框架：Tailwind CSS + Headless UI

**选择理由**：
- ✅ **快速开发**：utility-first，无需写大量 CSS
- ✅ **响应式设计**：`md:`, `lg:` 前缀轻松实现
- ✅ **深色模式**：`dark:` 前缀原生支持
- ✅ **Headless UI**：提供无样式组件（Modal, Dropdown 等）
- ✅ **与现有项目兼容**：可以保留部分 CSS Modules

**配置文件**：
```javascript

  content: ['./src/**/*.{js,ts,jsx,tsx}'],
  theme: {

      colors: {
        paper: 'var(--paper)',

        accent: 'var(--accent)',
      },
    },
  },

}
```

### 数据处理：gray-matter + Remark

**选择理由**：
- ✅ **gray-matter**：解析 Markdown Frontmatter
- ✅ **Remark**：Markdown 转 HTML，支持插件
- ✅ **Rehype**：HTML 后处理（代码高亮、图片优化）

**示例代码**：

## 核心模块设计

### 1. Vault 同步模块 (vault-sync.js)

**职责**：
- 扫描 Obsidian Vault 文件
- 解析 Markdown 和 Frontmatter
- 提取结构化数据
- 生成 JSON 文件

**核心函数**：

```javascript
/**
 * 主同步函数
 */
async function syncVault(vaultPath, outputPath) {
  console.log('🔄 开始同步 Vault...')
  
  // 1. 读取个人信息
  const profile = await parseProfile(vaultPath)
  
  // 2. 读取每日记录（最近 30 天）
  const dailyLogs = await parseDailyLogs(vaultPath, 30)
  
  // 3. 读取项目信息
  const projects = await parseProjects(vaultPath)
  
  // 4. 读取知识卡片
  const cards = await parseCards(vaultPath)
  
  // 5. 计算统计数据
  const stats = calculateStats(dailyLogs, projects, cards)
  
  // 6. 生成关系图谱
  const graph = generateKnowledgeGraph(cards)
  
  // 7. 输出 JSON 文件
  await writeJSON(`${outputPath}/vault-data.json`, {
    profile,
    dailyLogs,
    projects,
    cards,
    lastSync: new Date().toISOString()
  })
  
  await writeJSON(`${outputPath}/learning-stats.json`, stats)
  await writeJSON(`${outputPath}/knowledge-graph.json`, graph)
  
  console.log('✅ 同步完成！')
}
```

**数据结构示例**：

### 2. 数据加载模块 (lib/data.ts)

**职责**：
- 在 Next.js 页面中加载 JSON 数据
- 提供类型安全的数据访问
- 缓存数据以提升性能

**核心函数**：

```typescript
import fs from 'fs'
import path from 'path'
import type { VaultData, LearningStats } from '@/types/vault'

/**
 * 加载 Vault 数据
 */
export function loadVaultData(): VaultData {
  const dataPath = path.join(process.cwd(), 'data/vault-data.json')
  const jsonData = fs.readFileSync(dataPath, 'utf8')
  return JSON.parse(jsonData)
}

/**
 * 加载学习统计
 */
export function loadLearningStats(): LearningStats {
  const dataPath = path.join(process.cwd(), 'data/learning-stats.json')
  const jsonData = fs.readFileSync(dataPath, 'utf8')
  return JSON.parse(jsonData)
}

/**
 * 获取特定项目
 */
export function getProjectById(id: string) {
  const data = loadVaultData()
  return data.projects.find(p => p.id === id)
}

/**
 * 获取特定卡片
 */
export function getCardById(id: string) {
  const data = loadVaultData()
  return data.cards.find(c => c.id === id)
}

/**
 * 搜索卡片
 */
export function searchCards(query: string) {
  const data = loadVaultData()
  return data.cards.filter(c => 
    c.title.includes(query) || 
    c.content.includes(query) ||
    c.tags.some(t => t.includes(query))
  )
}
```

interface ProgressBarProps {
  current: number
  total: number
  label?: string
  color?: 'blue' | 'green' | 'purple'
}

export function ProgressBar({ 
  current, 
  total, 
  label,
  color = 'blue' 
}: ProgressBarProps) {
  const percentage = Math.round((current / total) * 100)
  
  return (
    <div className="progress-bar">
      {label && <div className="label">{label}</div>}
      <div className="bar-container">
        <div 
          className={`bar bar-${color}`}
          style={{ width: `${percentage}%` }}
        >
          <span className="percentage">{percentage}%</span>
        </div>
      </div>
      <div className="stats">
        {current} / {total}
      </div>
    </div>
  )
}

## 数据同步方案

### GitHub Actions 配置

```yaml
# .github/workflows/daily-sync.yml
name: Daily Vault Sync

on:
  schedule:
    # 每天北京时间 10:00 (UTC 02:00)
    - cron: '0 2 * * *'
  
  # 支持手动触发
  workflow_dispatch:

jobs:
  sync:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout Website Repo
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Checkout Vault Repo
        uses: actions/checkout@v4
        with:
          repository: your-username/my-brain
          path: vault
          token: ${{ secrets.VAULT_ACCESS_TOKEN }}
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Install Dependencies
        run: npm install gray-matter remark remark-html
      
      - name: Run Sync Script
        run: node scripts/vault-sync.js
        env:
          VAULT_PATH: ./vault
          OUTPUT_PATH: ./data
      
      - name: Commit Changes
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add data/
          git diff --quiet && git diff --staged --quiet || git commit -m "chore: sync vault data $(date +'%Y-%m-%d')"
          git push
      
      - name: Trigger Vercel Deployment
        run: echo "Vercel will auto-deploy on push"
```
